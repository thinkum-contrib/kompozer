Préférences de &brandShortName;

Des paramètres importants pour &brandShortName; sont définis dans la fenêtre des préférences. Vous pouvez définir vos préférences en fonction de vos besoins, et &brandShortName; les respectera à chaque fois que vous l'utiliserez.

Cette section décrit les paramètres du panneau de préférences de &brandShortName;. Si vous n'êtes pas actuellement dans le panneau des préférences, suivez les étapes suivantes :

  1. Ouvrez le menu Outils (menu &brandShortName; sur Mac OS X) et sélectionnez Préférences.
  2. Cliquez sur la catégorie qui vous intéresse.

Pour plus d'information sur les paramètres de publication de &brandShortName;, consultez la section Paramètres de publication.

Préférences générales

Les préférences de &brandShortName; vous permettent de spécifier les paramètres pour l'enregistrement des fichiers et l'édition de tableau. Ces paramètres s'appliquent à chaque document que vous créez.

Si vous n'êtes pas déjà en train de visionner les préférences de &brandShortName;, suivez les étapes suivantes :

  1. Ouvrez le menu Outils (menu &brandShortName; sur Mac OS X) et sélectionnez Préférences.
  2. Cliquez sur la catégorie Général.

Menu des pages récentes

Nombre maximal de pages listées : Spécifie le nombre maximal de pages listées dans le sous-menu Pages récentes du menu Fichier.

Lors de l'enregistrement ou de la publication des pages

  • Conserver le format source d'origine : Cette option conserve les sauts de lignes et le formatage original de la page pour le code source HTML. Sélectionnez ceci si vous voulez conserver les espaces (lignes vides, tabulations, etc.) qui rendent le code source HTML plus lisible. Cette préférence n'affecte pas l'affichage de vos pages dans un navigateur.
  • Reformater le source HTML : Cette option reformate le code source HTML pour le rendre plus lisible mais insère des sauts de lignes et des indentations. Cette préférence n'affecte pas l'affichage de vos pages dans un navigateur.
  • Enregistrer les images et les autres fichiers associés : Si cette option est cochée, toutes les images, les fichiers JavaScript (JS), les feuilles de styles en cascades (CSS), et les autres fichiers associés sont enregistrés dans un même emplacement quand le document est enregistré la première fois ou lorsqu'il est enregistré dans un nouvel emplacement. Si cette option est décochée, seul le fichier HTML est enregistré.

  • Par exemple, lorsque vous éditez une page distante, ce paramètre permet que tous les fichiers associés avec la page distante soient enregistrés localement quand vous enregistrez la page sur votre disque dur.

  • Toujours afficher la fenêtre du gestionnaire lors de la publication des pages : Si cette option est cochée, &brandShortName; affiche toujours la boîte de dialogue Publier une page quand vous cliquez sur le bouton Publier ou lorsque vous choisissez Publier dans le menu Fichier. Si cette option n'est pas cochée, &brandShortName; n'affiche la boîte de dialogue Publier une page que lorsqu'il aura besoin de plus d'informations afin de pouvoir publier la page.

Édition des tableaux

Conserver les dimensions du tableau lors de l'insertion ou de la suppression de cellules : Sélectionnez ceci si vous voulez que &brandShortName; conserve toujours les dimensions du tableau (c'est-à-dire, qu'il garde une forme rectangulaire) en ajoutant des cellules où c'est nécessaire. Si vous décochez cette option, quand vous supprimerez une ou plusieurs cellules, &brandShortName; supprimera la bordure de la cellule aussi, ce qui peut donner un tableau avec des espaces vides, ou un contour qui apparaît irrégulier à cause de cellules non appariées.

[ Retour au début de la section ]

Édition avec des feuilles de styles (CSS)

Utiliser les styles CSS au lieu des éléments et attributs HTML : Active l'utilisation de feuilles de style en cascades (CSS) pour le formatage dans vos documents &brandShortName;. Avec cette option, &brandShortName; génère du code HTML 4.01 avec des styles CSS intégrés pour les éléments.

Si cette préférence n'est pas activée, &brandShortName; génère du code HTML 4.01, mais n'utilise pas les styles CSS.

Comparé au code HTML, le code HTML avec formatage CSS est plus portable, plus maintenable, et plus compatible quand il est visionné avec différents navigateurs. Si vous activez cette préférence et que vous éditez un document créé sans CSS, &brandShortName; remplace les éléments édités avec des styles CSS.

Si vous activez les styles CSS, vous pouvez choisir une couleur de surlignage pour le texte sélectionné en utilisant le bouton de couleur de surlignage sur la barre de mise en forme. Vous pouvez également choisir une couleur de fond pour tout élément de la page. (Ces fonctionnalités ne sont pas disponibles si cette préférence n'est pas activée.)

Gestionnaire de sites

Toujours ouvrir un document dans un nouvel onglet : Sélectionnez ceci si vous voulez ouvrir les documents dans de nouveaux onglets quand vous utilisez le gestionnaire de sites. Si cette option est décochée, un double-clic sur un nom de fichier dans le gestionnaire de sites ouvrira le fichier dans une nouvelle fenêtre.

Polices

Vous pouvez changer les polices affichées par défaut dans &brandShortName;.

Polices pour l'encodage

Pour changer les polices par défaut :

  1. Depuis la liste déroulante Polices pour l'encodage, choisissez un encodage. Par exemple, pour définir les paramètres de police pour l'encodage Occidental, choisissez Occidental.
  2. Sélectionnez également la police proportionnelle pour le texte, qui devrait être de type sérif (par exemple Times New Roman) ou sans sérif (par exemple Arial). Ensuite, spécifiez la taille que vous souhaitez pour la police proportionnelle.
  3. Spécifiez la police que vous souhaitez utiliser pour les polices sérif, sans sérif, cursives, fantaisies et à largeur fixe. Vous pouvez également changer la taille des caractères pour les polices à largeur fixe.

Finalement, vous pouvez spécifier la taille minimale autorisée. Ceci peut être très utile sur les sites qui utilisent de très petites polices à peine lisibles.

Vous pouvez aussi spécifier la résolution d'affichage, qui est normalement de 96 dpi.

Paramètres des nouvelles pages

Les paramètres de pages vous permettent de spécifier les couleurs et les images de fond qui s'appliquent à chaque document que vous créez.

Si vous n'êtes pas déjà en train de visionner les paramètres de pages, suivez ces étapes :

  1. Ouvrez le menu Outils (menu &brandShortName; sur Mac OS X) et sélectionnez Préférences.
  2. Cliquez sur la catégorie Paramétres de pages.

Auteur

Entrez votre nom. Ceci ajoutera votre nom au code source HTML pour chaque nouvelle page que vous créez. Votre nom sera ajouté à la section <head> de votre page Web et ne sera pas visible sur la page.

[ Retour au début de la section ]

Balisage

  • Langage : Vous pouvez choisir le langage utilisé par défaut à la création d'un nouveau document. Vous avez le choix entre le HTML4.0 ou le XHTML1.0.
  • DTD : Vous pouvez choisir la définition de type de document (DTD) à utiliser à la création d'un nouveau document. Vous avez le choix entre une définition de transition ou stricte.

[ Retour au début de la section ]

Apparence par défaut des pages

  • Couleurs par défaut (celles du navigateur) : Sélectionnez ceci si vous voulez que vos pages utilisent les paramètres de couleurs du navigateur du lecteur pour le texte et les liens.
  • Utiliser des couleurs personnalisées : Sélectionnez ceci si vous voulez toujours spécifier les couleurs à appliquer au texte et aux liens. Puis, pour chaque élément, choisissez une couleur en cliquant sur le bouton de couleur situé à côté de chaque élément.
  • Image de fond : Tapez l'emplacement et le nom du fichier image, ou cliquez sur Parcourir… pour le rechercher sur votre disque dur ou sur le réseau.

    Note : Les images de fond sont empilées et masquent la couleur de fond.

  • Pour changer le nom de l'auteur sur une page individuellement : Ouvrez le menu Format et choisissez Titre et propriétés de la page….

    Pour changer les couleurs et l'image de fond sur une page individuellement : Ouvrez le menu Format et choisissez Couleurs et fond de page….

[ Retour au début de la section ]

Internationalisation

  • Langue : Choisissez la langue dans laquelle le document html est supposé être écrit.

  • Par exemple, si le document est écrit en français canadien, sélectionner French/Canada[fr-ca] indiquera au navigateur que le document est écrit en français canadien. Et le code source html contiendra <html lang="fr-ca"> comme indicateur de langue du document.

  • Sens d'écriture : Choisissez le sens d'écriture du texte en fonction de la langue que vous utilisez dans le document. Ceci résout les problèmes des utilisateurs qui créent des pages html dans des langues dont le sens d'écriture va de droite à gauche comme par exemple, l'arabe, l'urdu, etc.

  • Pour changer la spécification de langue pour une page individuellement : Ouvrez le menu Format, choisissez Titre et propriétés de la page… et changez la langue dans la section Internationalisation.

    Pour changer le sens d'écriture pour une page individuellement : Ouvrez le menu Format, choisissez Titre et propriétés de la page… et changez le sens d'écriture dans la section Internationnalisation.

  • Encodage : L'encodage détermine comment les octets représentant votre document HTML sont traduits en caractères lisibles. Avec cette option vous pouvez choisir l'encodage que vous voulez utiliser pour votre page Web. Par défaut, l'encodage ISO-8859-1 est utilisé, mais vous pouvez utiliser un autre choix, par exemple UTF-8 ou ISO-8859-15.

Options avancées

Connexion

Cette section décrit comment utiliser le panneau des préférences de proxies. Si vous nêtes pas déjà en train de le visionner, suivez ces étapes :

  1. Ouvrez le menu Outils (menu &brandShortName; sur Mac OS X) et sélectionnez Préférences.
  2. Cliquez sur la catégorie Avancées…, puis sur le bouton Paramètres… dans la sous-rubrique Connexion.

Le panneau des préférences de proxies vous permet de définir que &brandShortName; utilise un proxy :

Avant de commencer : Demandez à votre administrateur réseau s'il y a un fichier de configuration de proxy ou les numéros de port du proxy.

  • Connexion directe à Internet : Choisissez ceci si vous ne voulez pas utilisez de proxy.
  • Configuration manuelle du proxy : Choisissez ceci si vous n'avez pas de fichier de configuration de proxy (URL).
    • Proxy HTTP, Proxy SSL, Proxy FTP, Proxy Gopher : Tapez le nom ou l'adresse IP du serveur proxy. Tapez le numéro de port dans le champ Port.
    • Hôte SOCKS : Tapez le nom ou l'adresse IP du server proxy. Tapez le numéro de port dans le champ Port.
      • SOCKS v4, SOCKS v5 : Quand vous entrez un hôte SOCKS, sélectionnez SOCKS v4 ou SOCKS v5 en fonction de la version de SOCKS utilisée par votre proxy.
    • Pas de proxy pour : Tapez les domaines et/ou les adresses IP pour lesquelles vous ne voulez pas utiliser de proxy. Séparez chaque entrée par une virgule. (Exemple : .yourcompany.com, .yourcompany.co.nz, 192.168.1.0/24)
  • URL de configuration automatique du proxy : Choisissez ceci si vous avez un fichier de configuration ou une URL, puis tapez l'URL de configuration.

Comportement de la touche Entrée

L'option crée automatiquement un nouveau paragraphe définit le comportement de la touche Entrée. En choisissant ceci, cela crée un nouveau paragraphe si on appuie sur la touche Entrée. Sans cette option, des balises <br> sont ajoutées pour des retours chariots.

[ Retour au début de la section ]

Correction orthographique en temps réel

Cocher l'option Souligner les mots mal orthographiés activera la vérification orthographique en temps réel intégrée à &brandShortName; pour souligner les mots mal orthographiés, donnant par conséquent une indication visuelle à l'utilisateur pour revérifier l'orthographe.

[ Retour au début de la section ]

Caractères spéciaux

Cette option permet à l'utilisateur de choisir quels caractères spéciaux seront codés en entités par &brandShortName;. Choisissez l'une des options suivantes :

  • Uniquement & < > ' et les espaces insécables
  • Les lettres ci-dessus et Latin-1
  • Tout caractère ayant une entité équivalente dans la spécification HTML 4
  • Utiliser la notation &#..; pour tout caractère non-ASCII
