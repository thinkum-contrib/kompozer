Publier des pages Web avec &brandShortName;

Si vos pages HTML n'existent que sur votre disque dur local, vous seul pourrez les voir. &brandShortName; vous permet de publier vos pages sur un ordinateur distant appelé serveur Web.

Quand vous publiez vos pages sur un serveur Web, &brandShortName; copie (télécharge) vos pages vers un ordinateur qui permet à d'autres personnes de naviguer sur vos pages. La plupart des fournisseurs d'accès à Internet fournissent un espace sur leurs serveurs pour la publication de vos pages personnelles. Pour trouver un serveur Web, contacter votre fournisseur d'accès, la hotline ou l'administrateur système.

Publier un document

Pour publier un document :

  1. Ouvrez le document HTML que vous souhaitez publier ou créez un nouveau document &brandShortName;.
  2. Quand vous êtes prêt à publier le document à distance, cliquez sur le bouton Publier

    . Si vous aviez déjà publié ce document auparavant, &brandShortName; se rappelle des paramètres de publication du document et commence à le publier. Pendant la publication, &brandShortName; affiche une boîte de dialogue de l'état de publication.

    • Si vous n'avez jamais publié ce document auparavant, &brandShortName; affiche l'onglet Paramètres dans la boîte de dialogue Publier une page. Ainsi vous pourrez entrer les informations concernant le site de publication distant du document. Voir Paramètres de publication pour plus d'information. Quand vous avez fini d'entrer les informations, cliquez sur Publier.
    • Si vous n'avez jamais enregistré ce document auparavant, &brandShortName; affiche l'onglet Publier dans la boîte de dialogue Publier une page. Ainsi vous pourrez entrer le nom de fichier. Voir Publier une page pour plus d'information. Après avoir entré le nom du fichier, cliquez sur le bouton Publier.
  3. Pour afficher votre page dans le navigateur, cliquer sur le bouton Navigateur. Testez les liens de la page et assurez-vous qu'il n'y a aucune image manquante.
  4. Continuez à éditer la page tant que nécessaire. Quand vous êtes prêt à mettre à jour la page distante avec vos changements, cliquez sur le bouton Publier.

Quand vous publiez un document pour la première fois, &brandShortName; change l'URL file:/// du document en une URL http:// pour indiquer que vous éditez maintenant le document publié. Si vous désirez enregistrer le document localement (sur le disque dur de votre ordinateur), cliquez sur le bouton Enregistrer. &brandShortName; vous demandera de choisir un nom de fichier et un emplacement sur votre disque dur pour le document.

Astuces pour éviter les liens cassés ou les images manquantes

  • Assurez-vous que les noms de vos fichiers &brandShortName; se terminent bien par l'extension .html ou .htm.
  • Assurez-vous que les noms de vos fichiers images se terminent bien par l'extension .JPG, .GIF ou .PNG.
  • N'utilisez pas d'espaces ou d'autres caractères spéciaux (pas de lettres accentuées ou liées par exemple).
  • Utilisez des noms de fichiers courts tout en majuscules ou tout en minuscules et avec éventuellement des chiffres.
  • Si vos images apparaissent comme des liens cassés quand vous affichez une page située sur le serveur Web, c'est peut être que vous avez oublié d'inclure les images quand vous avez publié. Ouvrez le menu Fichier et choisissez Publier sous… pour afficher la boîte de dialogue Publier une page. Dans l'onglet Publication, assurez-vous que Inclure images et autres fichiers est bien coché puis cliquez sur Publier.

Pour plus d'astuces sur la résolution des problèmes, voir Résoudre les problèmes courants de publication..

Mettre à jour un document publié

Pour mettre à jour un document publié :

  1. Dans une fenêtre &brandShortName;, ouvrez le menu Fichier et choisissez Pages récentes, sélectionnez ensuite votre document dans la liste.
    Sinon, vous pouvez atteindre l'emplacement du fichier en entrant l'adresse HTTP du document (l'adresse Web du document) dans le champ de la fenêtre Ouvrir une page Web.
  2. Ouvrez le menu Fichier et choisissez Ouvrir une page Web….
  3. Editez le document autant que nécessaire.
  4. Quand vous êtes prêt à mettre à jour la page distante avec vos changements, cliquez sur le bouton Publier dans la barre d'outils de &brandShortName;.

Astuce : Pour supprimer une page ou une image que vous avez publiée sur un serveur Web, vous devez utiliser un programme de FTP. Vous devez aussi utiliser un programme de FTP si vous voulez créer des sous-dossiers ou renommer des fichiers sur un serveur Web. Demandez à votre hébergeur s'il recommande un programme de FTP particulier. Habituellement, vous pouvez trouver des informations sur les programmes de FTP sur le site Web de votre hébergeur, dans la section aide ou la section support. Des programmes de FTP sont aussi disponibles sur des sites de téléchargements comme ZDNET.

Créer un nouveau site de publication

Si vous projetez de publier sur plus d'un emplacement distant, vous pouvez configurer &brandShortName; pour enregistrer les informations de publication pour chaque site distant que vous utilisez. Ainsi vous n'aurez pas à saisir les paramètres à chaque fois que vous publierez.

Pour créer un nouveau site de publication, à partir d'une fenêtre &brandShortName; :

  1. Ouvrez le menu Édition et choisissez Paramètres des sites de publication…. &brandShortName; affiche la boîte de dialogue des Paramètres de publication.
  2. Cliquez sur Nouveau Site.
  3. Dans Nom du site, entrez l'intitulé que vous voudrez avoir quand vous vous référerez à ce site de publication.

    Par exemple, si vous souhaitez utiliser ce nouveau site de publication pour publier des documents en rapport avec un projet Meteor, vous voudrez peut être utiliser Meteor comme nom de site. Les noms de site vous aident à vous rappeler quels types de documents vous publiez sur chaque site.

  4. Dans Adresse de publication, entrez l'URL complète qui vous a été fournie par votre hébergeur. Cette url doit commencer par ftp:// ou par http://.

    L'adresse de publication spécifie l'emplacement où les documents sont publiés (envoyés) sur ce site. Si vous n'êtes pas certain des paramètres à saisir, contacter votre hébergeur.

  5. Dans Adresse HTTP de votre page d'accueil, entrez l'URL complète que vous entreriez dans votre navigateur pour accéder à vos pages. N'ajoutez pas de nom de fichier ou de sous-dossier à la partie URL.

    Cette URL doit toujours commencer par http://. Dans certains cas, cette URL peut être la même que l'adresse de publication. Si vous n'êtes pas certain des paramètres à saisir, contacter votre hébergeur ou bien laissez en blanc.

  6. Dans Identifiant, entrer le nom d'utilisateur que vous utilisez pour vous connecter chez votre hébergeur (par exemple avec votre logiciel FTP).
  7. Dans Mot de passe, entrez le mot de passe associé à votre identifiant.
  8. Sélectionnez Mémoriser le mot de passe pour enregistrer votre mot de passe de façon sécurisée. Ainsi vous n'aurez pas besoin de le redonner à chaque fois que vous voudrez publier des pages sur ce site.
  9. Cliquez sur OK.

Paramètres de publication

Cette section décrit les paramètres de publication de &brandShortName;. Pour une information sur les paramètres généraux et ceux d'une nouvelle page, voir Préférences de &brandShortName;.

Publier une page - Édition

L'onglet Édition dans Publier une page vous demande de spécifier où vous voulez publier le document. Ces paramètres ne s'appliquent qu'au document courant.

Si vous n'êtes pas déjà dans l'onglet Édition dans Publier une page, suivez ces étapes :

  1. Ouvrir le menu Fichier et choisissez Publier sous…. La boîte de dialogue Publier une page apparaît.
  2. Cliquer sur l'onglet Édition.
  • Nom du site : Liste tous les sites de publication que vous avez créés, ainsi vous pouvez choisir l'emplacement où vous voulez publier votre document. Pour créer un nouveau site, cliquer sur le bouton Nouveau site.
  • Titre de la page : Spécifie le titre de la page tel qu'il apparaît dans la barre de titre du navigateur quand vous regardez la page dans un navigateur. Le titre de la page apparaît aussi dans votre liste de marque-pages, si vous marquez cette page.
  • Nom de fichier : Spécifie le nom de fichier du document. Assurez-vous d'inclure l'extension .html ou .htm dans le nom du fichier.

  • Attention : Si un fichier sur le site distant où vous publiez a le même nom qu'un document que vous envoyez, ce fichier envoyé remplacera le fichier distant existant. Il n'y aura pas de demande de confirmation.

  • Sous-dossier du site pour cette page : Si vous laissez ceci en blanc, &brandShortName; publie la page dans le sous-dossier de publication principal (racine) de ce site. Si vous voulez publier la page dans un sous-dossier distant, situé dans le dossier de publication principal, entrez le nom de ce dossier ou bien choisissez-le dans la liste. &brandShortName; garde en mémoire les dossiers que vous avez déjà tapés dans cette case, ainsi vous pouvez choisir dans une liste de dossiers distants que vous avez déjà utilisés. Rappelez-vous que les noms des sous-dossiers sont sensibles à la casse.

  • Note : Le sous-dossier du site que vous choisissez doit déjà exister sur le site distant.

  • Inclure images et autres fichiers : Si cette case est cochée, &brandShortName; publie toutes les images et autres fichiers référencés par la page. Vous pouvez choisir de publier ces fichiers dans le même dossier que la page ou dans un sous-dossier qui existe déjà dans le dossier principal.

Astuce : Pour créer des sous-dossiers distants ou effacer des pages ou des images, vous devez utiliser un programme de FTP. Demandez à votre hébergeur, s'il recommande un programme de FTP particulier. Habituellement, vous pouvez trouver des informations sur les programmes de FTP sur le site Web de votre hébergeur, dans la section aide ou la section support. Des programmes de FTP sont aussi disponibles sur des sites de téléchargements comme ZDNET.

Publier une page - Paramètres

L'onglet Paramètres dans Publier une page vous permet de spécifier vos informations de connexion pour le site de publication distant ainsi que les paramètres de publication pour ce site. Ces paramètres s'appliquent au document courant et à tous les documents que vous publierez sur ce site.

Si vous n'êtes pas déjà dans l'onglet Paramètres dans Publier une page , suivez ces étapes :

  1. Ouvrir le menu Fichier et choisissez Publier sous…. La boîte de dialogue Publier une page apparaît.
  2. Cliquer sur l'onglet Paramètres.
  • Nom du site : Spécifie l'intitulé que vous voulez utiliser pour désigner ce site de publication. Utilisez un nom court qui vous aidera à identifier ce site de publication.
  • Adresse de publication : Spécifie l'URL complète donnée par votre hébergeur. Cette URL doit commencer soit par ftp:// soit par http://. Cette adresse est souvent mentionnée comme le nom d'hôte ou le nom d'hôte du serveur.

  • L'adresse de publication spécifie l'emplacement où les documents sont publiés (envoyés) sur ce site. Si vous n'êtes pas sûr de ce que vous devez mettre, demandez à votre hébergeur.

  • Adresse HTTP de votre page d'accueil : Spécifie l'adresse complète du dossier principal de publication. C'est l'adresse Web de votre page principale sur le site Web. Elle ne doit pas inclure de nom de fichier ou de sous-dossier comme partie de l'URL.

  • Cette URL doit toujours commencer par http://. Dans certains cas, cette URL est la même que celle de l'adresse de publication. Si vous n'êtes pas certain des paramètres à saisir, contactez votre hébergeur ou bien laissez en blanc.

  • Identifiant : Spécifie le nom d'utilisateur que vous utilisez pour vous identifier chez votre fournisseur d'accès ou sur le réseau.
  • Mot de passe : Spécifie le mot de passe de votre identifiant.
  • Mémoriser le mot de passe : Sélectionner ceci pour chiffrer et sauvegarder de façon sécurisée votre mot de passe en utilisant le gestionnaire de mot de passe, ainsi vous n'aurez pas à le redonner à chaque fois que vous publierez des pages sur ce site.

Résolution des problèmes habituels de publication

Si un de vos fichiers ne peut pas être publié, la boîte de dialogue État de la publication affiche un message d'erreur qui peut vous aider à déterminer ce qui s'est mal passé et comment le résoudre. Cliquez sur le bouton Détails dans la boîte de dialogue État de la publication pour avoir de l'aide pour résoudre votre problème de publication.

Si vous n'arrivez toujours pas à publier le fichier, sauvegardez-le sur votre disque dur en ouvrant le menu Fichier de &brandShortName; et choisissez Enregistrer. Vous pouvez ensuite ouvrir le fichier plus tard et réessayer de le publier. Pour le localiser rapidement plus tard, ouvrez le menu Fichier de &brandShortName;et choisissez Fichiers récents.

Vérifiez vos paramètres de publication

Pour vérifier vos paramètres de publication :

  1. Fermez la boîte de dialogue État de publication si elle est ouverte.
  2. Ouvrez le menu Édition et choisissez Paramètres des sites de publication….
  3. Dans la boîte de dialogue des paramètres de publication, assurez-vous que les paramètres du site où vous êtes en train de publier sont corrects. Si vous n'en êtes pas sûr, contactez votre fournisseur d'accès ou le service d'hébergement de votre site Web.
    • Vérifiez que vous avez saisi correctement les paramètres de publication. Vous pourriez avoir accidentellement fait une faute de frappe.
    • Vérifiez que vous avez saisi l'adresse de publication correcte.Les services d'hébergement ou les fournisseurs d'accès désignent parfois l'adresse de publication par nom du serveur, le nom d'hôte ou serveur/hôte. Ils fournissent souvent un emplacement de publication du type ftp.monfai.com/monlogin, où monlogin est votre identifiant.

      Pour que l'adresse de publication soit correcte, elle doit être précédée soit de ftp:// soit de http://. Par exemple, pour le site sus-mentionné, l'adresse correcte de publication serait ftp://ftp.monfai.com/monlogin.

Vérifiez vos noms de fichiers

Examinez les noms de tous les fichiers que vous ne parvenez pas à publier :

  • Utilisez seulement des nombres, des lettres minuscules ou des lettres majuscules. Même s'il est acceptable que les noms de fichiers contiennent des majuscules, vous éviterez de possibles erreurs plus tard, quand vous chercherez le fichier publié si vous n'utilisez que des minuscules dans vos noms de fichiers.

    Quand vous publiez un fichier sur un serveur Web, les noms de fichiers deviennent sensibles à la casse. Il peut s'avérer difficile de vous souvenir quels fichiers n'ont que des minuscules et lesquels ont un mélange de majuscules et de minuscules.

    Par exemple, quand vous tentez de localiser un fichier publié en entrant l'adresse Web du fichier dans la barre d'adresse de votre navigateur, vous devez entrer le nom tel que vous l'avez créé, avec la même combinaison de majuscules et minuscules.

  • N'utilisez pas de caractères de ponctuations ou d'espaces. Les soulignements ( _ ) et traits d'union ( - ) sont autorisés.
  • La fin de votre nom de fichier doit être .html ou .htm (pour les noms de fichiers de &brandShortName;).
  • Utilisez moins de 32 caractères.

Résoudre les erreurs de publication

Si un ou plusieurs de vos fichiers ne peut pas être publié, regardez les messages que &brandShortName; affiche dans l'espace de l'état de publication de la boîte de dialogue de publication. Vous pouvez vous servir de ces messages d'erreurs pour vous aider à déterminer ce qui s'est mal passé et pour résoudre ce problème.

Message d'erreur

nom_du_fichier non trouvé

ou

X de Y fichiers n'ont pas pu être publiés

Description de l'erreur : Une ou plusieurs images ou feuilles CSS n'ont pas pu être publiés parce que &brandShortName; ne les a pas trouvés. Les raisons les plus fréquentes peuvent être :

  • L'emplacement du fichier que vous avez tapé est incorrect.
  • L'emplacement du fichier sur le Web est inaccessible.
  • L'emplacement du fichier a été changé ou le fichier a été effacé ou déplacé vers un autre emplacement.

Solutions possibles :

  • Cherchez les images cassées dans la page que vous essayez de publier. Les images cassées apparaissent avec cette icône dans la page. Pour corriger l'adresse de l'image, double-cliquez sur l'image cassée pour afficher la boîte de dialogue Propriétés de l'image afin de pouvoir saisir l'adresse correcte.
  • Enlevez l'image cassée de la page en la sélectionnant (cliquez une fois sur l'image) puis appuyez sur la touche Retour arrière ou Suppr. sur votre clavier.
  • Si l'image n'est pas disponible parce que le serveur où l'image est hébergée est inaccessible, réessayez de publier cette page plus tard.
  • Si le fichier manquant est un fichier CSS, vous devez d'abord vérifier que l'emplacement du fichier CSS est correct. Pour corriger l'adresse du fichier dans &brandShortName;, cliquez sur l'onglet HTML Source et modifiez l'emplacement du fichier dans le code HTML. Vous ne devriez éditer le source HTML que si vous êtes familier avec les balises HTML.

Message d'erreur :

Le sous-dossier nom_du_répertoire n'existe pas sur ce site ou le nom de fichier nom_du_fichier est déjà utilisé par un autre répertoire

ou

Le nom de fichier nom_du_fichier est déjà utilisé par un autre répertoire

Description de l'erreur : Vous avez spécifié un nom de sous-dossier distant qui n'existe pas sur le site de publication . &brandShortName; peut seulement publier dans un sous-dossier distant qui existe déjà sur le site de publication. Ou alors vous avez spécifié un nom de fichier qui est identique au nom d'un sous-dossier qui existe déjà sur le site de publication.

Par exemple, dans la boîte de dialogue Publier une page, dans l'onglet Édition :

  • Dans la case Sous-dossier du site pour cette page, il est possible que vous ayez tapé le nom d'un sous-dossier qui n'existe pas sur le site de publication.
  • Vous avez coché Inclure les images et autres fichiers, puis vous avez tapé le nom d'un sous-dossier qui n'existe pas sur le site de publication.
  • Un des fichiers que vous tentez de publier a le même nom qu'un sous-dossier sur le site de publication.

Solutions possibles :

  • Utilisez un logiciel FTP séparé si vous voulez créer, renommer ou effacer des sous-dossiers sur le site de publication. Demandez à votre hébergeur s'il recommande un programme de FTP particulier. Habituellement, vous pouvez trouver des informations sur les programmes de FTP sur le site Web de votre hébergeur, dans la section aide ou la section support. Des programmes de FTP sont aussi disponibles sur des sites de téléchargements comme ZDNET.
  • N'utilisez pas de noms de sous-dossiers qui finissent par .html ou .htm. Seuls vos fichiers créés par &brandShortName; devraient finir par .html ou .htm.
  • Les noms de sous-dossiers sont sensibles à la casse, assurez-vous donc de saisir le nom du sous-dossier exactement tel qu'il apparaît sur le site de publication.

Message d'erreur :

Le serveur n'est pas accessible. Vérifiez votre connexion et réessayez plus tard.

Description de l'erreur : Cette erreur peut avoir de multiples causes. Par exemple :

  • Vos paramètres de publication sont peut-être erronés.
  • Votre connexion à Internet a peut être cessé.
  • Votre modem ou tout autre équipement que vous utilisez pour vous connecter à Internet ne fonctionne peut-être pas correctement.
  • Le serveur Web sur lequel vous tentez de publier est peut-être inaccessible à cause d'un problème technique ou d'une circonstance inconnue.
  • Votre fournisseur d'accès ou votre hébergeur a peut-être des problèmes techniques.

Solutions possibles :

  • Vérifiez que vos paramètres de publication sont corrects et que vous les avez saisis correctement. Voir Vérifiez vos paramètres de publication pour plus d'information.
  • Assurez-vous que vous êtes bien connecté à Internet en essayant, par exemple, de regarder une page dans votre navigateur. Essayez de contacter http://www.mozilla.org/ par exemple.
  • Si votre connexion Internet ne fonctionne pas, vérifiez que tout votre matériel, connexion téléphonique, modem et connexion réseau fonctionnent correctement.
  • Utilisez votre navigateur pour essayer de voir une page du site sur lequel vous essayez de publier. Si vous arrivez à afficher d'autres sites Web mais pas celui sur lequel vous voulez publier, c'est sans doute que votre hébergeur rencontre des problèmes techniques.
  • Essayez de publier plus tard, votre hébergeur ou le serveur Web rencontre peut-être des problèmes techniques temporaires.

Message d'erreur :

Vous n'avez pas la permission de publier à cet emplacement.

Description de l'erreur : Vous essayez de publier sur un emplacement que vous n'êtes pas autorisé à utiliser. Vous pouvez seulement publier sur des emplacements où vous avez été autorisé par un hébergeur ou un fournisseur d'accès.

Solutions possibles :

  • Vérifiez que l'identifiant et le mot de passe que vous avez entrés dans la boîte de dialogue Paramètres de publication ou dans l'onglet Édition de la boîte de dialogue de publication sont corrects.
  • Contactez votre hébergeur pour savoir où vous pouvez publier vos pages sur leur site.
  • Trouvez un hébergeur afin de pouvoir publier vos pages. Dans votre navigateur, cherchez hébergement de site web, par exemple.
  • La plupart des fournisseurs d'accès à Internet fournissent gratuitement un espace Web pour publier vos pages personnelles.

Message d'erreur :

L'espace disque est insuffisant pour enregistrer le fichier nom_du_fichier.

Description de l'erreur : Le disque dur du serveur distant est plein, ou bien vous avez dépassé le quota disque alloué par votre hébergeur.

Solutions possibles :

  • Utilisez un logiciel FTP indépendant pour effacer les fichiers inutiles sur votre site de publication.
  • Voyez avec votre hébergeur s'il est possible d'augmenter l'espace disque qui vous est alloué, ou bien changez d'hébergeur pour en trouver un dont les offres correspondent mieux à vos besoins. Ou bien répartissez vos dossiers sur plusieurs espaces web (chez plusieurs hébergeurs par exemple).
  • Si le serveur Web est situé dans votre entreprise ou votre établissement scolaire, contactez l'administrateur réseau pour voir si vous pouvez publier sur un emplacement différent qui a plus d'espace libre, ou si vous pouvez obtenir plus d'espace disque sur votre emplacement actuel.

Message d'erreur :

Le nom du fichier ou du sous-dossier est trop long.

Description de l'erreur : Le serveur ne supporte pas le nombre de caractères contenus dans le nom du fichier ou du sous-dossier.

Solutions possibles :

  • Utilisez moins de 32 caractères pour vos noms de fichier. En effet, certains systèmes d'exploitation ne supportent pas les noms de fichier de plus de 32 caractères.
