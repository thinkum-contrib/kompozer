Az &brandShortName; beállításai

A &brandShortName; számos fontos beállítása a Beállítások panelen változtatható meg. ó felhasználók szükségleteiknek megfelelően állíthatják be az &brandShortName; programot. A beállítások felhasznákra lebontva tárolódnak.

Ez a fejezet az &brandShortName; beállításait ismerteti. A megfelelő párbeszédpanel a következő lépésekkel jeleníthető meg:

  1. Válasszuk az Eszközök > Beállítások menüt (Mac OS X alatt a Mozilla menüt) a Beállítások panel megnyitásához.
  2. Kattints a kategóriákra, hogy lásd a megfelelő beállításokat.

A közzétételi beállítások megváltoztatásáért tekints el ide.

Általános beállítások

Az általános beállítások megengedik, hogyí finomíts az állományokon és a táblázatokon. Ezek az összes létrehozott dokumentumra érvényesek lesznek.

Amennyiben még nem látod a beállítások rengetegét, kövesd a következő lépéseket:

  1. Válasszuk a Szerkesztés menü (Mac OS X alatt a Mozilla menü) Beállítások parancsát.
  2. A fülekből válaszd ki az Általános feliratú fület.

Legutóbb szerkesztett oldalak

A listában megjelenő oldalak maximális száma: adjuk meg a Fájl menü Legutóbb szerkesztett oldalak parancsának választásakor megjelenő dokumentumok számát.

Fájlok mentésekor

  • Retain original source formatting: Akkor érdemes választani, ha meg szeretnénk tartani a HTML forráskód jobb olvashatóságát biztosító üres helyeket (üres sorok, tabulátorok stb.) A beállítás nincs hatással az oldalak böngészőben való megjelenítési módjára.
  • HTML forrás újraformázása: ez a beállítás újraformázza a HTML forráskódot: üres sorok és behúzások beszúrásával olvashatóbbá teszi azt. A beállítás nincs hatással az oldalak böngészőben való megjelenítési módjára.
  • Képek és más csatolt fájlok mentése az oldalak mentésekor: bejelölésekor az összes kép, JavaScript (JS), stíluslap (CSS) és egyéb kapcsolódó fájl is mentésre kerül a dokumentummal azonos helyre, amikor először vagy új helyre mentjük a dokumentumot. Ha üresen hagyjuk a négyzetet, csak a HTML fájlt menti a rendszer.

  • Ha például távoli oldalt szerkesztünk, ez a beállítás biztosítja, hogy a kapcsolódó fájlok helyi mentése is megtörténjék, amikor a merevlemezre mentjük az oldalt.

  • Always show Publish dialog when publishing pages: Amennyiben be van kapcsolva, akkor a program egyfolytában mutatni fogja a közzétételi ablakot.

Cascading Style Sheets(CSS) Editing

Használni a CSS dolgokat a HTML dokumentumban: Megengedi a CSS használatát, a dokumentum formázásában. Ezzel az eszközzel &brandShortName; generál egy HTML 4.01 formázást, amelyben minden stíluselemet a CSSből vesz.

Amennyiben ez az opció nincs megengedve &brandShortName; generál HTML 4.01-es formázást, de CSS elemek nélkül.

A stíluslapokat használó formázás jobban átvihető a rendszerek között, karbantartása egyszerűbb és kompatibilis a különböző böngészőkkel. A beállítás engedélyezésekor a stíluslapok nélkül létrehozott dokumentum módosításakor a HTML-szerkesztő stílusokkal helyettesíti a szerkesztett elemeket. Amennyiben engedélyezed ezt az opciót, a szerkesztett elemek már CSSben lesznek megoldva.

A stílusok engedélyezésekor a kijelölt szöveg kiemeléséhez a Formázó eszköztár szövegkiemelésre szolgáló gombja használható. Ugyanígy színes háttér is választható az oldal tetszőleges eleméhez. (Ezek a szolgáltatások a beállítás tiltásakor nem állnak rendelkezésre.)

Betűk

Beilleszthetjük saját vagy más betűtípust is, de ügyeljünk arra, hogy a kliensen is megtalálható legyen ez a font.

Fonts for

A Fonts for:-ban meg lehet definiálni font-családokat. Hogy ezt megtedd, kövesd a lépéseket:

  1. Válassza ki a Kódolás legördülő menüből a karakterkódolást. Hogy az alapértelmezett dolgokat tegye be, válaszd ki a Western-t.
  2. Jelöld ki a szöveget és határozd meg a relatív méretét.
  3. Válassz a következő fontcsaládok közül: Serif, Sans-serif, Cursive Fantasy vagy Monospace.

Meghatározhatod a minimum betűméretet. Ez hasznos a kis betűmérettel rendelkező oldaknál.

Meghatározhatod még a felbontást is, ami alapból 96 dpi.

Haladó beállítások

Kapcsolat

Ez a témakör a Proxy panellapot ismerteti. Amennyiben nem ismered ezt a lehetőséget, kövesd a küövetkező lépéseket:

  1. Válasszuk a Szerkesztés menü (Mac OS X alatt a &brandShortName; menü) Beállítások parancsát.
  2. A Haladó kategóriában kattintsunk a Proxy elemre. (Ha nem láthatóak a kategória beállításai, a lista kibontásához kattintsunk duplán a Haladó kategóriára.)

A proxy beállítások lehetővé teszik a proxi használatát:

Mielőtt hozzákezdenénk: kérdezzük meg a rendszergazdát, hogy tartozik-e konfigurációs fájl a proxykiszolgálóhoz, illetve tudjuk meg a kiszolgálók nevét és portszámát.

  • Direkt kapcsolat: Használjuk ezt ha nincs proxynk.
  • Kézi proxy konfiguráció: Ha nincs saját proxyd.
    • HTTP Proxy, SSL Proxy, FTP Proxy, Gopher Proxy: Adjuk meg a proxy szerver ip címét. Adjuk meg a portot a Port mezőben.
    • SOCKS-kiszolgáló: írjuk be a proxykiszolgáló nevét vagy numerikus IP-címét. Adjuk meg a portot a Port mezőben.
      • SOCKS v4, SOCKS v5: When entering a SOCK Host, select "SOCKS v4" or "SOCKS v5," depending on what version of SOCKS is used for the proxy.
    • Ne használjon proxyt az alábbiaknál: adjuk meg azokat a tartományokat, illetve IP-címeket, amelyeknél nem kívánunk proxykiszolgálót használni. Több bejegyzés vesszővel elválasztva adható meg, például: .vallalat.hu, .vallalat.com, 192.168.1.0/24.
  • Automatikus proxybeállítás URL: ezt akkor válasszuk, ha a proxykiszolgálóhoz konfigurációs fájl tartozik, és írjuk be URL-címét.

Return Key Behaviour

The option "Return in a paragraph always creates new paragraph" defines the behaviour of the return key. Choosing it creates a new paragraph if you hit the "Return" key while typing in a paragraph. Without this option <br> tags are added for carriage returns.

Különleges karakterek

This options allow the user to choose which special characters are coded as entities by &brandShortName;. Válassz egyet a következők közül:

  • Csak & < > ' és non-breakable üreshelyek
  • A fentiek és a Latin-1 betűk
  • Minden karakternek legyen ekvivalense a HTML 4.0-ban
  • &#..; jelölés használata minden nem ASCII-karakterre
