Publicar páginas web con &brandShortName;

Si las páginas sólo existen en su disco duro local, podrá verlas de forma privada. &brandShortName; le permite publicar sus páginas web en un ordenador remoto llamado servidor web.

Cuando usted publica sus páginas en un servidor web, &brandShortName; copia (sube) sus páginas a un ordenador remoto que permite que otras personas puedan navegar por sus páginas. Muchos ISPs (Internet Service Provider; Proveedores de Servicios de Internet) proporcionan espacio en sus servidores para poder publicar páginas web. Para encontrar un servidor web en el que publicar sus páginas, pregunte a su ISP, centro de atención a usuarios o al administrador de sistema.

Publicar un documento

Para publicar un documento:

  1. Abra el archivo HTML que quiere publicar, o cree un nuevo documento en &brandShortName;.
  2. Cuando esté listo para publicar un documento de forma remota, haga clic en el botón Publicar .

    Si ha publicado el documento anteriormente, &brandShortName; recuerda las opciones de publicación y publica el documento con esas opciones. Mientras se esté llevando a cabo la publicación, &brandShortName; mostrará un mensaje con el estado.

    • Si nunca ha publicado un documento, &brandShortName; muestra las propiedades en el cuadro de diálogo Publicar página para que pueda introducir información acerca del documento remoto. Vea Publicar páginas - Configuración para más información. Cuando haya terminado de introducir la información, pulse en Publicar.
    • Si nunca ha guardado el documento, &brandShortName; mostrará la pestaña Publicar en el cuadro de diálogo Publicar página, para que pueda introducir el nombre de archivo del documento. Vea Publicar página - Publicar para más información. Después de introducir el nombre, haga clic en Publicar.
  3. Para ver su página publicada, haga clic en el botón Navegar. Pruebe los enlaces de la página y compruebe que no falta ninguna imagen.
  4. Continúe editando la página según necesite. Cuando esté listo para volver a subir la página con los cambios, haga clic en el botón Publicar.

Cuando se publica un documento por primera vez, &brandShortName; cambia el inicio de la URL del documento de file:/// a http:// para indicar que ahora está editando el documento en el servidor web. Si quiere guardar el documento localmente (en el disco duro de su ordenador), haga clic en el botón Guardar. Se le pedirá que elija un nombre y ubicación de archivo en su disco duro para el documento.

Consejos para evitar enlaces rotos o imágenes perdidas

  • Asegúrese de que los archivos creados con &brandShortName; terminan en .html o .htm. Asegúrese de que sus imágenes terminan en .JPG, .GIF, o .PNG. No use espacios u otros caracteres especiales en el nombre de los archivos. Guarde los archivos con un nombre corto y utilice sólo minúsculas o mayúsculas y números.
  • Si sus imgenes aparecen como enlaces rotos cuando navega por el servidor web, puede que haya olvidado incluir las imágenes cuando ha publicado la página. Abra el menú Archivo y elija Publicar como… para mostrar el diálogo Publicar página. En la pestaña Publicar, asegúrese de seleccionar Incluir imágenes y otros archivos y luego pulse Publicar.

Para ver mas consejos, vea Resolver problemas comunes de publicación.

Actualizar un documento ya publicado

Para actualizar un documento ya publicado:

  1. En una ventana de &brandShortName;, abra el menú Archivo y elija Páginas Recientes, y después seleccione el documento de la lista.

    Alternativamente, navegue hasta la dirección del documento que quiera actualizar introduciendo la dirección HTTP (la dirección web del documento) en la barra de direcciones del navegador.

  2. Abra el menú Archivo y elija Editar página.
  3. Edite el documento según sea necesario.
  4. Cuando esté listo para actualizar la página remota con sus cambios, haga clic en Publicar en la barra de herramientas de &brandShortName;.

Consejo: para eliminar una página o una imagen que ha publicado en un servidor web, debe utilizar un programa FTP (File Transfer Protocol). También deberá utilizar un programa FTP si quiere crear subdirectorios o renombrar archivos en el servidor web. Pregunte a su ISP si recomienda algún programa FTP en especial. Normalmente podrá encontrar información sobre programas FTP en las secciones de ayuda o soporte de la web de su ISP. Los programas FTP también están disponibles en algunos sitios de descargas, como Softonic.

Cambiar el nombre del archivo o el directorio de publicación

Para cambiar el nombre de archivo o el directorio de publicación del documento:

  1. En una ventana de &brandShortName;, abra el menú Archivo, elija Páginas recientes y seleccione el documento en la lista.

    Alternativamente, navegue hasta el directorio del documento que quiere actualizar introduciendo la dirección HTTP (la dirección web de la página) en el campo Dirección del navegador.

  2. Abra el menú Archivo y elija Editar página.
  3. Edite el documento según sea necesario.
  4. Abra el menúArchivo de &brandShortName; y elija Publicar como…. &brandShortName; mostrará la pestaña Publicar en la ventana Publicar página.
  5. Introduzca un título diferente para la página, si es necesario.
  6. Introduzca un nombre diferente para la página, si es necesario.
  7. De la lista de sitios, elija la ubicación de publicación que quiera usar. Para crear una nueva ubicación de publicación, elija Nuevo sitio. Vea Publicar página - Configuración para más información.
  8. Haga clic en Publicar para guardar el documento en su nueva ubicación.

Crear un nuevo sitio de publicación

Si piensa publicar documentos en más de un sitio remoto, puede configurar &brandShortName; para guardar la información de la publicación de cada sitio remoto que use, de manera que no tenga que introducir la información cada vez que quiera publicar.

Para crear un nuevo sitio de publicación, comience desde una ventana de &brandShortName;:

  1. Abra el menúEditar y elija Configuración del sitio de publicación. &brandShortName; mostrará la ventana Configuración de publicación.
  2. Haga clic en Nuevo sitio.
  3. Para Nombre del sitio elija el nombre que identificará el sitio remoto.

    Por ejemplo, si usará el nuevo sitio para publicar documentos relativos al proyecto Meteor, puede querer usar el nombre de sitio Meteor. Los nombres de sitio le recuerdan el tipo de documentos que publica en cada sitio.

  4. Para Dirección de publicación, introduzca la URL completa proporcionada por su ISP, administrador del sistema o servicio de hospedaje web. Esta URL debe empezar bien por ftp://, bien por http://.

    La dirección de publicación especifica la ubicación donde van a ser publicados (subidos) los documentos en el sitio. Si no está seguro de qué es lo que tiene que introducir, pregunte a su ISP o adminitrador del sistema.

  5. En la Dirección HTTP de su página inicial, introduzca la URL completa que introduciría en la barra de navegación para ver páginas web en este sitio. No incluya un nombre de archivo o directorio como parte de esta URL.

    Esta URL siempre debe empezar por http://. En algunos casos, esta URL es la misma que la dirección de publicación. Si no está seguro de qué introducir, pregunte a su ISP o administrador del sistema o deje el campo en blanco.

  6. Como nombre de usuario introduzca el nombre que use para iniciar sesión en su ISP o servicio de hospedaje web.
  7. Como contraseña introduzca la contraseña para su nombre de usuario.
  8. Seleccione Guardar contraseña para guardar su contraseña de forma segura usando el administrador de contraseñas para que no tenga que introducirla cada vez que vaya a publicar en el sitio web.
  9. Haga clic en Aceptar.

Cambiar el sitio predeterminado de publicación

Si ha configurado más de un sitio de publicación, pero normalmente sólo utiliza uno de ellos, puede designarlo como el sitio predeterminado para la publicación de archivos. &brandShortName; usará el sitio predeterminado de publicación para publicar todos los documentos que quiera subir, a menos que específicamente elija un sitio alternativo.

Sin importar cuántos sitios haya configurado, siempre puede publicar un documento en otro sitio eligiendo Publicar como en el menú Archivo de &brandShortName;. Vea Cambiar el nombre de archivo o el directorio de publicación para más información.

Para elegir el sitio de publicación predeterminado, comience en la ventana principal de &brandShortName;:

  1. Abra el menú Editar, y elija Configuración del sitio de publicación. &brandShortName; mostrará la ventana Configuración de publicación.
  2. Seleccione un sitio de publicación de la lista.

    Si sólo tiene un sitio de publicación, &brandShortName; lo usará como el predeterminado.

  3. Haga clic en Seleccionar como valor predeterminado.
  4. Haga cic en Aceptar para confirmar sus cambios.

Eliminar un sitio de publicación

Eliminar un sitio de publicación elimina la configuración del sitio en &brandShortName;. Si más tarde desea publicar de nuevo en ese sitio, deberá introducir otra vez la configuración para dicho sitio.

Para borrar la configuración de un sitio, empiece en la ventana principal de &brandShortName;:

  1. Abra el menú Editar, y elija Configuración del sitio de publicación. &brandShortName; mostrará la ventana Configuración de publicación.
  2. Seleccione un sitio de publicación de la lista.
  3. Haga clic en Eliminar sitio.

    &brandShortName; sólo elimina la configuración del sitio; el sitio remoto en sí no se ve afectado por este cambio.

  4. Haga clic en Aceptar para confirmar sus cambios.

Configuración de publicación

Esta sección describe la configuración de publicación de &brandShortName;. Para información sobre la configuración general y de nueva página de &brandShortName;, vea Preferencias de &brandShortName;.

Publicar página - Publicar

La pestaña Publicar, dentro de Publicar página, &brandShortName; le permite especificar dónde quiere publicar un documento. Esta configuración se aplicará al documento actual.

Si no está viendo la pestaña Publicar, siga estos pasos:

  1. Abra el menú Archivo y elija Publicar como. Aparecerá la ventana Publicar página.
  2. Haga clic en la pestaña Publicar.
  • Nombre del sitio: muestra todos los sitios que haya creado, para que pueda elegir uno de ellos para publicar su documento actual. Para crear un nuevo sitio, haga clic en Nuevo sitio.
  • Título de la página: especifica el título del documento que aparecerá en la barra de título de su navegador cuando vea la página en él. El título de la página también aparecere en su lista de marcadores si añade aquélla a éstos.
  • Nombre de archivo: especifica el nombre del archivo. Asegúrese de que su documento termina en .htm o .html.

  • Atención: si hay un archivo en el sitio remoto de igual nombre que el documento que va a publicar ahora, el documento remoto será reemplazado por el actual, sea cual sea el contenido del archivo remoto. Además, no se le pedirá confirmación acerca de esta acción.

  • Subdirectorio del sitio para esta página: si deja esto en blanco, &brandShortName; publicará la página en el directorio principal (raíz) del sitio. Si quiere que se publique en un subdirectorio que reside dentro del sitio, introduzca la ruta del subdirectorio o elíjala de la lista. &brandShortName; guardará la ruta que haya establecido, por lo que en posteriores publicaciones puede seleccionar fácilmente el subdirectorio de la lista. Tenga en cuenta que los subdirectorios de los sitios remotos son sensibles al uso de mayúsculas o minúsculas.

  • Nota: el subdirectorio que haya elegido debe existir en el sitio remoto.

  • Incluir imágenes y otros archivos: si está seleccionado, &brandShortName; publicará cualquier imagen o archivo referenciado por esta página. Puede elegir publicarlas en el mismo directorio que la página, o bien ubicarlas en otro subdirectorio distinto dentro del sitio.

Consejo: para crear o eliminar subdirectorios remotos, es necesario usar un programa FTP (File Transfer Protocol; Protocolo de Transferencia de Archivos). Pregunte a su ISP si recomienda algún programa FTP en especial. Normalmente podrá encontrar información sobre programas FTP en las secciones de ayuda o soporte del proveedor de servicios web. Los programas FTP también están disponibles en algunos sitios de descargas, como Softonic.

Publicar página - Configuración

En la pestaña Configuración, dentro de la ventana Publicar página, puede establecer los datos para identificarse en el sistema remoto (ordenador donde se van a subir los archivos para su publicación), así como la configuración de publicación para el sitio remoto. Esta configuración se aplicará al documento actual y a cualquier otro que utilice este sitio.

Si no está viendo la pestaña Configuración dentro de Publicar página, siga estos pasos:

  1. Abra el menú Archivo y elija Publicar como. Aparecerá la ventana Publicar página.
  2. Haga clic en la pestaña Configuración.
  • Nombre del sitio: especifique el nombre que quiere usar para este sitio de publicación. Introduzca un nombre corto, ya que esto le ayudará a identificar ese sitio.
  • Dirección de publicación: especifica la URL completa proporcionada por su ISP o administrador del sistema. Esta URL debería comenzar por ftp:// o http://. Este nombre es también conocido como "servidor web" o "nombre del host".

  • La dirección de publicación especifica el directorio donde se van a publicar (subir) los archivos de su ordenador. Si no está seguro de qué es lo que tiene que introducir, contacte con su ISP o su administrador del sistema.

  • Dirección HTTP de su página inicial: especifica la dirección completa de su directorio inicial de publicación. Ésta es la dirección web que se introducirá en los navegadores para ver la página inicial de su sitio web. No incluya ningún nombre de archivo o subdirectorio en este campo.

  • Esta URL debe comenzar siempre por http://. En algunos casos, esta URL es la misma que la dirección de publicación. Si no está seguro de qué es lo que tiene que introducir, contacte con su ISP o su administrador del sistema o deje el campo en blanco.

  • Nombre de usuario: introduzca el nombre que se utilizará para identificarse en su ISP o red local.
  • Contraseña: especifica la contraseña para su nombre de usuario.
  • Guardar contraseña: seleccione esta opción para cifrar y guardar su contraseña de manera segura en el administrador de contraseñas para no tener que introducirla siempre que quiera publicar algún archivo.

Solucionar problemas comunes de publicación

Si se produce un error con uno o más de sus archivos al tratar de publicarlos, el cuadro de diálogo de estado de publicación muestra un mensaje de error que puede ayudarle a determinar qué sucedió y cómo corregirlo. Pulse el botón Solución de problemas en el cuadro de diálogo para obtener ayuda sobre cómo solucionar su problema de publicación.

Si aún así no puede publicar un archivo, guarde el archivo en su disco duro abriendo el menú Archivo de &brandShortName; y escogiendo Guardar. Así podrá abrir más tarde el archivo y tratar de publicarlo. Para localizar rápidamente el archivo posteriormente, abra el menú Archivo y escoja Páginas recientes.

Verificar su configuración de publicación

Para verificar su configuración de publicación:

  1. Cierre el cuadro de diálogo de estado de la publicación, si está abierto.
  2. Abra el menú Editar y escoja Configuración de publicación.
  3. En el cuadro de diálogo de configuración de publicación confirme que los parámetros del sitio son correctos para el sitio en el que está intentando publicar. Si no está seguro, compruébelo con su ISP o servicio de hospedaje web.

Comprobar los nombres de sus archivos

Examine los nombres de los archivos que no se hayan publicado correctamente. Asegúrese de que los nombres de archivo:

  • Usan sólo números o letras minúsculas o mayúsculas. Aunque es correcto crear nombres de archivo que usen letras en mayúsculas, puede evitar posibles errores de localización de los archivos publicados si utiliza sólo letras minúsculas en sus nombres de archivo.

    Cuando publica archivos en un servidor web, los nombres de archivo diferencian mayúsculas y minúsculas. Puede que le resulte más difícil recordar nombres de archivo que utilizan sólo letras mayúsculas o que usen una mezcla de letras en minúsculas y mayúsculas.

    Por ejemplo, cuando intenta localizar un archivo publicado escribiendo la dirección web del archivo en el campo Dirección de Navigator, debe introducir el nombre de archivo exactamente tal y como lo creó, usando la misma combinación de letras minúsculas y mayúsculas.

  • No use signos de puntuación o espacios. Los subrayados ( _ ) y guiones ( - ) no presentan problemas.
  • Termine los archivos con .html o .htm (para los nombres de archivo de &brandShortName;).
  • Use menos de 32 caracteres.

Corregir errores de publicación

Si uno o más de sus archivos no se publica correctamente, mire los mensajes que &brandShortName; muestra en el área de estado de publicación del cuadro de diálogo. Puede utilizar estos mensajes de error para ayudarle a determinar qué falló y qué hacer para corregir el problema.

Mensaje de error:

Nombrearchivo no encontrado

o

X de Y archivos no pudieron ser publicados

Descripción del error: uno o más archivos de imágenes o CSS no pudieron ser publicados porque &brandShortName; no pudo encontrarlos. Algunas causas habituales pueden ser:

  • La ubicación del archivo que indicó no es correcta.
  • La dirección del archivo en la web no es accesible.
  • La dirección del archivo ha cambiado o el archivo ha sido eliminado o trasladado a otra dirección.

Posibles soluciones:

  • Busque imágenes rotas en la página que intenta publicar. Las imágenes rotas aparecerán con este icono en la página. Para corregir la dirección de la imagen, haga doble clic en la imagen rota para mostrar el cuadro de diálogo de propiedades de la imagen para que pueda introducir la dirección correcta.
  • Elimine la imagen rota de la página seleccionándola (pulse una vez sobre ella con el botón izquierdo) y a continuación pulse Eliminar o Suprimir en el teclado.
  • Si la imagen no está disponible porque el servidor en el que ésta reside no está accesible, pruebe a publicar la página más tarde.
  • Si el archivo que falta es un archivo CSS, primero debe verificar la dirección correcta del archivo CSS. Para corregir la dirección del archivo en &brandShortName;, pulse la pestaña Etiquetas HTML y edite la dirección del archivo en el código fuente HTML. Sólo debe editar el código fuente HTML si está familiarizado con las etiquetas HTML.

Mensaje de error:

El subdirectorio nombredirectorio no existe en este sitio o el nombre de archivo nombrearchivo ya está en uso por otro subdirectorio

o

El nombre de archivo nombrearchivo ya está en uso por otro subdirectorio

Descripción del error: ha especificado el nombre de un subdirectorio remoto que no existe en el sitio de publicación. &brandShortName; sólo puede publicar a un subdirectorio remoto que ya exista en la dirección de publicación. O bien ha especificado un nombre de archivo que es idéntico al nombre de un subdirectorio existente en el sitio de publicación.

Por ejemplo, en el cuadro de diálogo Publicar página, bajo la pestaña Publicar:

  • para Subdirectorio del sitio para esta página, puede que haya escrito el nombre de un subdirectorio que no existe en la dirección de publicación.
  • ha marcado Incluir imágenes y otros archivos y luego ha escrito el nombre de un subdirectorio que no existe en la dirección de publicación.
  • uno de los archivos que está intentando publicar tiene el mismo nombre que un subdirectorio del sitio de publicación.

Posibles soluciones:

  • Use un cliente FTP por separado si quiere crear, renombrar o eliminar subdirectorios en el sitio de publicación. Pregunte a su proveedor de servicios si recomienda un cliente FTP en particular. Normalmente podrá encontrar información sobre clientes FTP en las secciones de ayuda o soporte del sitio web de su proveedor de servicios. Los programas FTP también están disponibles en sitios shareware como Softonic.
  • No use nombres de subdirectorio que terminen en .html o .htm. Sólo los archivos de &brandShortName; deberían terminar en .html o .htm.
  • Los nombres de subdirectorio distingues mayúsculas y minúsculas, así que asegúrese de que introduce los nombres de subdirectorio exactamente tal y como aparecen en la dirección de publicación.

Mensaje de error:

El servidor no está disponible. Compruebe su conexión e inténtelo más tarde

Descripción del error: este error puede tener muchas causas, como por ejemplo:

  • Sus parámetros del sitio de publicación pueden no ser correctos.
  • Su conexión a Internet puede haberse cortado.
  • su modem u otro equipamiento que utilice para conectar a Internet podría no estar funcionando correctamente.
  • El servidor web en el que está intentando publicar podría no estar disponible debido a un problema técnico o a una circunstancia desconocida.
  • Su ISP o servicio de hospedaje web podría estar experimentando problemas técnicos.

Posibles soluciones:

  • Verifique que sus parámetros de publicación son correctos y que los ha introducido correctamente. Vea Verificar sus parámetros de publicación para más información.
  • Asegúrese de que su conexión a Internet funciona intentando ver una página usando Navigator. Por ejemplo, confirme que puede acceder correctamente a la página http://www.mozilla.org.
  • Si su conexión a Internet no funciona, verifique que todo el hardware, conexiones telefónicas, modems y conexiones de red están funcionando correctamente.
  • Use Navigator para intentar ver una página en el sitio web en el que está intentando publicar. Si puede ver con éxito otros sitios web pero no puede ver páginas en el sitio de publicación, su ISP o servicio de hospedaje web puede estar sufriendo problemas técnicos.
  • Intente publicar de nuevo más tarde. Su ISP, servicio de hospedaje web, o el servidor web pueden estar experimentando dificultades técnicas temporalmente.

Mensaje de error:

No tiene permiso para publicar en esta dirección.

Descripción del error: está intentando publicar en una dirección que no está autorizado a usar. Sólo puede publicar en sitios donde se le haya concedido acceso por su ISP o servicio de hospedaje web.

Posibles soluciones:

  • Verifique que ha introducido el nombre de usuario y contraseña correctos en el cuadro de diálogo de configuración de publicación, o en la pestaña Publicar del cuadro de diálogo Publicar.

Mensaje de error:

No hay suficiente espacio libre en disco para guardar el archivo nombrearchivo.

Descripción del error: el disco duro del servidor remoto está lleno, o puede haber excedido la cantidad de espacio en disco asignado a su cuenta por su ISP o servicio de hospedaje web.

El nombre del archivo o subdirectorio es demasiado largo.
