Tämän sanaston on tarkoitus toimia ainoastaan yksinkertaisena tietolähteenä. Sanaston termien, niihin liittyvien yksityisyys- ja turvallisuuskäsitteiden tai näiden johdannaisten määritelmät eivät ole täydellisiä eikä niitä tule tulkita auktoritatiivisina.

Sanasto

Apuohjelma
Mikä tahansa ohjelma, jolla avataan tai katsellaan &brandShortName;:lla ladattua tiedostoa. Liitännäiset ovat erityisiä apuohjelmia, jotka asentuvat Plugins-kansioon &brandShortName;:n ohjelmakansiossa. Microsoft Word, Adobe Photoshop, ja monet muut ohjelmat ovat apuohjelmia mutta eivät liitännäisiä, koska ne eivät toimi liitännäisten tapaan &brandShortName;:n ohjelmaikkunassa.
Asiakas
Asiakas (mm. Internet-selain) on ohjelma, joka lähettää pyyntöjä palvelimelle ja vastaanottaa siltä vastauksia. Palvelinohjelma toimii yleensä toisella tietokoneella. Tietokonetta, joka suorittaa asiakasohjelmaa, kutsutaan myös asiakkaaksi.
Cascading Style Sheets (CSS)
CSS on yksinkertainen tapa lisätä www-dokumentteihin tyylejä: mm. kirjasinlajeja, värejä sekä asetteluja. CSS mahdollistaa hypertekstisivun ulkoasun täydellisen määrittelyn.
Digitaalinen tunnus
Katso varmenne.
Extensible Markup Language (XML)
Avoin standardi tietojen kuvailemiseen. Toisin kuin HTML-kieli, XML sallii omien tagien luomisen. Lisätietoja saat standardointiorganisaatio W3C:n dokumentista Extensible Markup Language (XML).
File Transfer Protocol (FTP)
FTP on standardoitu yhteyskäytäntö tiedostojen siirtämiseen kahden, verkon välityksellä yhdistetyn tietokoneen kesken. Etätietokone, FTP-palvelin, vastaanottaa pyyntöjä toisilta, asiakkaiksi kutsutuilta tietokoneilta. Asiakkaat voivat ladata tiedostoja palvelimelta tai siirtää niitä sinne riippuen käyttöoikeuksista. FTP-yhteyskäytännössä on kuitenkin tietoturva-aukko, sillä siinä salasanat lähetetään selväkielisenä tekstinä, toisin kuin SFTP-yhteyskäytännössä. &brandShortName;:n sivustonhallinta käyttää FTP-yhteyskäytäntöä.
FIPS PUBS 140-1
Federal Information Processing Standards Publications (FIPS PUBS) 140-1 on Yhdysvaltain hallituksen standardi kryptografisille moduuleille. Kryptografinen moduuli on laite tai ohjelma, joka salaa ja purkaa salattuja tietoja tai suorittaa muita kryptografisia toimintoja, kuten luo tai todentaa varmenteita. Useiden Yhdysvaltain hallitukselle myytyjen tuotteiden täytyy täyttää standardin vaatimukset osittain tai kokonaan.
Hakukone
Www-ohjelma, jolla haetaan tietoa Internetistä yleensä kirjoittamalla hakusanoja verkkosivulla olevaan hakukenttään. Tällä hetkellä suosituin hakukone on Google. Hakukoneet keräävät tietojaan monilla eri tavoilla ja yleensä eri hakukoneet antavatkin eri tuloksia hauille.
Hypertext Markup Language (HTML)
HTML on www-sivuissa käytetty rakenteenkuvailukieli. HTML-standardissa määritellään sivujen kirjoittamiseen vaadittavat tagit, jotka kertovat tekstin rakenteen sekä lisäävät sivulle kuvia ja muita elementtejä.
Internet
Maailmanlaajuinen tietokoneverkko, jossa tietokoneet kommunikoivat keskenään standardoitujen yhteyskäytäntöjen, kuten TCP/IP, välityksellä. Internet kehitettiin alun perin Yhdysvaltain asevoimissa mutta on erityisesti World Wide Webin kehittämisen jälkeen kasvanut voimakkaasti niin käyttötarkoituksiltaan kuin käyttömahdollisuuksiltaankin.
IP-osoite
TCP/IP-verkossa olevan tietokoneen osoite. Jokaisella Internetiin liitetyllä tietokoneella on oma osoitteensa. Asiakkailla on joko kiinteä IP-osoite tai dynaaminen osoite, joka vaihtuu joka kerta, kun yhteys Internetiin muodostetaan.
Java
Sun Microsystemsin kehittämä ohjelmointikieli. Java-ohjelman erikoisuus on, että se voidaan muuttamatta suorittaa useilla erilaisilla tietokoneilla. Jos Java on asennettu tietokoneelle, &brandShortName; voi automaattisesti ladata ja suorittaa Java-ohjelmia (eli sovelmia).
JavaScript
JavaScript on komentosarjakieli, jota käytetään www-sivuilla yleisesti. JavaScriptillä sivuista tehdään usein vuorovaikutteisia. JavaScriptiä voi käyttää yhdessä Javan kanssa, mutta ne ovat eri kieliä. Javan ei tarvitse olla asennettuna, jotta JavaScript toimisi.
Julkaiseminen
Dokumenttien kopioimista www-palvelimelle kutsutaan julkaisemiseksi. Muut voivat käyttää asiakasohjelmiaan dokumenttien tarkasteluun vasta, kun dokumentit on julkaistu. &brandShortName;:ssa dokumentit julkaistaan sivustonhallinnan avulla.
Kryptografia
Tieteen ala, joka tutkii tapoja muuttaa tieto näennäisesti sotketuksi (salatuksi) ja takaisin selkokieliseksi (salauksen purku). Kryptografiaa tarvitaan esimerkiksi salattaessa tietoa, jota siirretään Internet-kauppojen tai pankkien palvelimille.
Liitännäinen
Erityinen apuohjelma, joka lisää uusia ominaisuuksia &brandShortName;:hun. Tällaisia ominaisuuksia ovat esimerkiksi kyky toistaa ääni- ja videotiedostoja selaimessa. Toisin kuin muut apuohjelmat, liitännäiset asentuvat suoraan &brandShortName;:n ohjelmankansion Plugins-kansioon, ja niitä voi käyttää suoraan &brandShortName;:n ohjelmaikkunasta. Esimerkiksi Macromedian Flash Player ja Sunin Java ovat molemmat liitännäisiä.
Navigaatiopalkki
Selaimen ikkunan yläosassa oleva työkalupalkki, jossa on tärkeimmät selaamiseen tarvittavat painikkeet kuten Edellinen ja Seuraava.
Palvelin
Palvelin on ohjelma, joka vastaanottaa pyyntöjä asiakkailta ja lähettää niille vastauksia (kuten toimittaa www-sivuja). Asiakasohjelma toimii yleensä toisella tietokoneella. Tietokonetta, joka suorittaa palvelinohjelmaa, kutsutaan myös palvelimeksi.
PKCS #11
Julkisen avaimen salausmenetelmän standardi, joka koskee älykorttien kaltaisia turvalaitteita.
Purkaminen (salauksen)
Salatun materiaalin muuntamista selkokielelle tai muuten ymmärrettävään muotoon kutsutaan (salauksen) purkamiseksi. Katso myös salaaminen.
Pääsalasana
Salasana, jonka voit asettaa suojaamaan kaikkia muita salasanojasi ja muita yksityistietojasi. &brandShortName; kysyy pääsalasanaa, kun sen suojaamia tietoja tarvitaan. Jos omistat useita turvalaitteita, jokaiselle turvalaitteelle täytyy asettaa oma pääsalasanansa.
Salaaminen
Salattaessa tietoa muutetaan niin, että sen sisältöä ei voi selvittää. Salaamalla tehdään viestien ja datan joutuminen vääriin käsiin erittäin vaikeaksi. Muiden kuin tiedon lähettäjän ja vastaanottajan on erittäin vaikeaa muuttaa viestiä selkokielelle eli purkaa sitä, koska tämä onnistuu vain, jos hallussa on oikea avain (kuten oikealla vastaanottajalla).
Salainen avain
Julkisen avaimen salausmenetelmän avainparin toinen avain. Salainen avain nimensä mukaisesti pidetään salassa ja sillä kyetään purkamaan salaus tiedoista, jotka on salattu avainparin julkisella avaimella.
Salasanojen hallinta
Halutessasi &brandShortName; tallentaa joidenkin tai kaikkien sivustojen tai palveluiden tarvitsemat käyttäjätunnukset ja salasanat tietokoneellesi siten, että käyttäessäsi sivustoa tai palvelua &brandShortName; syöttää tunnukset automaattisesti.
Secure Shell File Transfer Protocol (SFTP)
Secure Shell File Transfer Protocol (SFTP) on muutoin samankaltainen yhteyskäytäntö kuin FTP, mutta siinä yhteys on koko ajan salattu ja siten turvallisempi.
Secure Sockets Layer (SSL)
Yhteyskäytäntö, jossa voidaan todentaa sekä palvelin, että asiakasohjelma ja salata verkkoliikenne kumpaankin suuntaan. SSL-yhteys toimii TCP/IP-verkoissa HTTP-, LDAP-, IMAP-, NNTP- ja muiden korkean tason yhteyskäytäntöjen kanssa. Internet Engineering Task Forcen (IETF) uusi yhteyskäytäntö Transport Layer Security (TLS) perustuu SSL:ään. Katso myös todennus ja salaus.
Sivustonhallinta
&brandShortName;:n osa, joka hallitsee ohjelmalla luotujen dokumenttien lataamista ja palvelimelle siirtämistä. Sivustonhallinta siirtää dokumentit www-palvelimelle FTP-yhteyskäytännön välityksellä.
Suojattu sivusto
Sivusto, joka salaa palvelimen ja selaimen välisen verkkoliikenteen ja siten estää sivullisia seuraamasta liikennettä. Suojatut sivustot erottaa selaimessa yleensä jostakin erityisestä merkinnästä.
Joillakin sivustoilla on sekä suojattua että suojaamatonta sisältöä. Suojaamaton sisältö on yleensä yleisluontoista eikä siten tarvetta salaukselle ole. Yleensä selaimet merkitsevät myös tällaiset sivut jollakin tapaa.
Tilarivi
&brandShortName;:n ikkunan alareunassa näkyvä palkki, joka näyttää kohdistinta ympäröivät tagit.
TLS
Katso Secure Sockets Layer (SSL).
Todentaminen
Todentamalla varmistetaan toimijan henkilöllisyys tietoverkoissa. Yleisiä todentamismenetelmiä ovat salasanat, varmenteet tai PIN-koodit.
Transmission Control Protocol/Internet Protocol (TCP/IP)
Unix-yhteyskäytäntöpari, joka mahdollistaa yhteydet hyvin erilaisia käyttöjärjestelmiä ja verkkoja hyödyntävien tietokoneiden väillä. TCP/IP on Internetin toiminnan kannalta välttämätön järjestelmä, ja siitä on muodostunut maailmanlaajuinen standardi.
Uniform Resource Locator (URL)
Standardoitu osoitemuoto, jonka perusteella &brandShortName; voi hakea tiedostoja tai muita resursseja Internetistä tai paikalliselta tietokoneelta. URL:ää käytetään, paitsi selaimen osoitepalkkiin kirjoitettuna, myös www-sivujen linkeissä, ja sitä kutsutaan usein vain Internet- tai www-osoitteeksi.
Varmenne
Varmenne on todistus, jolla luotettu kolmas osapuoli vahvistaa, että sähköisessä allekirjoituksessa mukana oleva julkinen avain kuuluu tietylle henkilölle, yritykselle tai organisaatiolle. Sähköposti tai muu data allekirjoitetaan digitaalisesti käyttäen yksityistä avainta, joka on varmenteessa olevan julkisen avaimen vastinkappale.
Väliaikaistiedostot
Väliaikaistiedostot ovat tietokoneen kiintolevylle tai muistiin tallennettuja www-sivuja. &brandShortName; kerää väliaikaistiedostoja, kun selaat sivuja. Kun avaat sivun napsauttamalla linkkiä tai kirjoittamalla osoitteen, &brandShortName; vertaa mahdollista väliaikaiskopiota alkuperäiseen ja lataa tiedoston verkosta vain jos muutoksia on tapahtunut, mikä pienentää käsittely- ja latausaikoja.
Välityspalvelin
Ohjelma, joka toimii välikätenä palvelimen ja asiakasohjelman välisessä verkkoliikenteessä. Välityspalvelin välittää eri asiakasohjelmien pyyntöjä palvelimille.
W3C
World Wide Web Consortium (W3C) on yhteenliittymä, joka luo World Wide Webin standardisuosituksia. W3C:n mukaan "The World Wide Web Consortiumin tarkoituksena on varmistaa www:n eri osien yhteensopivuus erilaisten tekniikoiden avulla (määritykset, suositukset, ohjelmistot ja työkalut) ja siten parantaa sen suorituskykyä." W3C:n suosituksia, mm. HTML 4.01, XHTML 1.1 sekä CSS 2.1, kutsutaan usein www-standardeiksi.
World Wide Web
Se osa Internetiä, joka koostuu www-palvelimiin tallennetuista sivuista, ja jota voi tarkastella www-selaimiksi kutsutuilla asiakasohjelmilla. Tunnetaan myös nimillä www, web ja webbi.
Www-palvelin
Www-palvelin on tietokone, joka ajaa www-palvelinohjelmistoa (kuten Apache). Www-palvelin välittää asiakkaille erilaisia www-dokumentteja ja on vastuussa siitä, että pyydettyä www-sivustoa voi tarkastella Internet-selaimessa.
Www-sivu
Yksittäinen World Wide Webin dokumentti, jolla on ainutkertainen osoite (URL). Www-sivut voivat sisältää tekstiä, hyperlinkkejä sekä grafiikkaa.
Www-sivusto
Kokoelma toisiinsa hyperlinkeillä liittyviä, saman aihepiirin www-sivuja, jota ylläpitää jokin yhtiö, organisaatio tai henkilö. Www-sivusto voi sisältää tekstiä, grafiikkaa, ääni- ja videotiedostoja sekä linkkejä muille www-sivustoille.
