あ行

アドレス [address]
　→IPアドレス
　→URL
暗号化 [encryption]
内容を隠すために情報を「かき混ぜる」(スクランブル) 過程。例えばコンピュータ間の接続を暗号化すれば、接続中に流れる情報を第三者が元に戻す (復号化) するのは困難になる。暗号化された情報は、適切な鍵を保有する人間のみが復号化できる。
暗号解読法 [cryptography]
情報を「かき混ぜ」(スクランブル、暗号化)、「もとに戻す」(解読、復号化) 技術及び手段。例えば、商用ウェブサイトと利用者のブラウザ間で受け渡す情報の暗号化・復号化に暗号技術が用いられる。
インターネット [The Internet]
TCP/IPのような標準の通信手順で無数のコンピュータが通信を行っている、世界規模のネットワーク。最初は1969年にアメリカ軍向けに開発されたが、教育・研究機関も接続されるようになり、1990年代後半には多数の企業、団体、個人も参加するようになった。今日ではインターネットは電子メール、WWWの閲覧、インスタントメッセージ、ニュースグループなど多種多様に用いられる。
ウェブページ [web page]
World Wide Web上の単一の文書。独自のアドレスもしくはURLにより指定され、文章、ハイパーリンク、画像を含む。
ウェブサイト [web site]
ハイパーリンクにより繋がり、単一の企業、団体、個人により運営される、一連のウェブページ群。ウェブサイトは文章、画像、音声、動画ファイル、そして他のウェブサイトへのリンクを含む。
ウェブサーバ [web server]
ウェブサーバとは、サーバソフトウェア (例えばApache）により実現され、様々な種類のウェブ文書をクライアントへと提供する。我々がウェブサイトをブラウザで表示できるのは、要求されたデータを送信してくれるウェブサーバというソフトウェアのおかげである。

か行

キャッシュ [cache]
ハードディスクやRAMに保存された、ウェブページのコピーのこと。ウェブページを閲覧すると&brandShortName;はそのコピーを溜める。リンクをクリックしたりURLを入力して、キャッシュにコピーが保存されているページを表示しようとすると、まず&brandShortName;はコピーとオリジナルを比較する。更新されていなければキャッシュを使って表示し、ダウンロード時間が節約できる。
クライアント [client]
サーバに要求を送信し、情報を受け取るソフトウェア(ブラウザソフトなど)。通常は別のコンピュータ上で動作する。クライアントソフトが動作するコンピュータもクライアントと称される。
検索エンジン [search engine]
WWWから特定の情報を検索することができる、ウェブベースのプログラム。検索エンジンは、ウェブ文書の全文やキーワードの一覧を検索したり、ウェブ文書の調査や検索のために手作業での索引作成に図書館員を使用する。一般的に、検索エンジンの入力欄に語句 (クエリと呼ばれる) を入力すると、関連するウェブページの一覧が表示される。
公開 [publishing]
ウェブ文書をウェブサーバにアップロードするプロセスを公開と呼んでいる。ウェブ文書をウェブサーバに公開して初めて、他の人が彼らのクライアントを用いて文書の閲覧が出来るようになる。&brandShortName;では、サイトマネージャが公開作業を担当する。

さ行

サイトマネージャ
&brandShortName;が作成した文書のアップロード、ダウンロードの管理をする&brandShortName;の機能。ウェブサーバとのウェブ文書の転送にはFTPを用いている。
サーバ [server]
通常別のコンピュータ上で動作するクライアントから、要求を受け、情報を送信する、ウェブページを供給するようなソフトウェア。サーバのソフトウェアが動作するコンピュータもサーバと称される。
証明書 [certificate]
デジタル版の身分証明書のようなもの。証明書は、個人、会社あるいは他のものの名前を指定し、公開暗号鍵 (それは証明書に含まれている) がそれに属することを証明する。あなたがメッセージあるいは他のデータにデジタル署名するとき、そのメッセージ用のデジタル署名は、あなたの証明書中の公開暗号鍵と一致する、秘密鍵によって作成される。
ステータスバー [Status Bar]
&brandShortName;ウィンドウの下部にあるツールバー。現在カーソルがある部分のタグの入れ子状態が表示される。
セキュア・ファイル転送プロトコル (SFTP) [Secure Shell File Transfer Protocol]
セキュア・ファイル転送プロトコル (Secure Shell File Transfer Protocol, SFTP) はFTPと類似のものであるが、セッションを暗号化することにより安全性をより増したものになっている。

ツールチップ [tooltip]
特定の場所の上にマウスカーソルを置いたときに現れる、文字を含む小さな枠。マウスカーソルを置いた場所の説明が含まれていることが多い。
デジタルID [digital ID]
　→証明書

な行

ナビゲーションツールバー [Navigation Toolbar]
&brandShortName;の画面上部にある、戻る、進むボタンなどを含むツールバーのこと。
認証 [authentication]
パスワード、証明書、暗証番号 (PIN)、その他の情報のコンピュータネットワークでの同一性を証明するもの。

は行

ファイル転送プロトコル (FTP) [File Transfer Protocol]
ファイル転送プロトコルは、コンピュータのファイルをネットワーク経由で転送するときに用いられる、標準のプロトコルである。外部コンピュータ (FTPサーバ) が、クライアントと呼ぶコンピュータとの接続を受け入れることで始まる。接続が一旦確立すると、クライアントがファイルを取得したり、特権さえあればファイルをサーバに転送することが出来る。ただしFTPには、パスワードが (暗号化されずに) 平文として転送されるという、セキュリティホールがある。これを修正したSFTPというものもある。&brandShortName;のサイトマネージャは、ファイルの公開 (アップロード) 時にFTPを利用する。
パスワードマネージャ [Password Manager]
名前とパスワードをコンピュータのハードディスク上に保存し、再訪問時に入力することを可能にする&brandShortName;の機能。
秘密鍵 [private key]
公開鍵暗号方式で用いられる対の暗号鍵のうちの一方。秘密鍵は秘密にされ、対応する公開鍵で暗号化されたデータを復号化するのに用いられる。
復号化 [decryption]
暗号化された情報を元に戻すこと。暗号化も参照。
不正表示 [misrepresentation]
個人や団体がそうでないと提示すること。例えば、あるウェブサイトが家具屋のふりをしてクレジットカードでの支払いを受けるが、品物を送らないこと。偽装も参照。
プラグイン [plug-in]
ヘルパーアプリケーションの1種で、&brandShortName;で音楽やビデオクリップの再生ができるようにするもの。一般のヘルパーアプリケーションと異なり、プラグインは&brandShortName;がインストールされているディレクトリ下にインストールされ、&brandShortName;の画面内で開かれる。例えば、音声プラグインは音声ファイルをウェブページ内や電子メール中で再生する。有名なプラグインとしてはMacromedia Flash PlayerやJavaがある。
プロキシ [proxy]
サーバとクライアント両方の役割をする、その他のクライアントの利益となるよう転送要求を代行する仲介・媒介ソフトウェア。
ヘルパーアプリケーション [helper application]
&brandShortName;でダウンロードしたファイルを開いたり表示したりするアプリケーション。プラグインとは、&brandShortName;をインストールしたディレクトリ下のPluginsディレクトリにインストールされ、&brandShortName;内で開くことのできる、ヘルパーアプリケーションの一種。Microsoft WordやAdobe Photoshopなど、外部アプリケーションはヘルパーアプリではあるがプラグインではない。何故なら&brandShortName;のディレクトリ下にインストールされるわけではないが、ダウンロードダイアログから起動することが出来るからである。
保護されたサイト [secure site]
悪意あるインターネットユーザの転送内容の盗聴を妨害するために、&brandShortName;との通信に暗号を用いているサイト。保護されたサイトを訪問すると、&brandShortName;のステータスバーとロケーションバーに鍵アイコンが表示される。またそのサイトのドメイン名がステータスバーに表示 (悪意あるサイトがあなたのデータを盗むのを妨げる為に) され、ロケーションバーの背景が黄色で表示されるようになる。
中にはデータを保護するものとしないものを併用するサイトがある。保護されないデータは多くの場合個人的なものではないが、常にそうであるとはいえない。この時、斜線付きの鍵アイコンがステータスバー及びロケーションバーに表示されるが、ステータスバーにサイトのドメインは表示されない。このような場合は、現在表示中のウェブページが完全に保護されている訳ではないということが判るわけである。

ま行

マスターパスワード [master password]
証明書マネージャがマスターキーや秘密鍵をセキュリティデバイスに保存するときに使用するパスワードのこと。これらのデータを利用するときにマスターパスワードが必要となる。セキュリティデバイスごとに異なるマスターパスワードが必要。

英字

CSS [Cascading Style Sheet]
CSSとは、スタイル (例えばフォント、色、配置など) をウェブ文書に適用する、単純な機構のこと。CSSにより、ハイパーテキストのスタイルを完全に、総合的に制御することが出来る。
FIPS PUBS 140-1
米国連邦情報処理標準 (Federal Information Processing Standards Publications、FIPS PUBS) 140-1とは、アメリカ政府が標準と定める暗号モジュール (暗号化・復号化その他暗号処理 (デジタル署名の作成・証明など) に用いられるハードウェアやソフトウェア) の実装である。アメリカ政府機関に納入する製品はFIPS標準に沿う必要がある。
FTP [File Transfer Protocol]
　→ファイル転送プロトコル
HTML [Hypertext Markup Language]
ウェブページに使用される文書の書式。HTMLの標準仕様は、ウェブページ作成に使用される、文書レイアウト、フォント、スタイル、画像などの要素のタグを定義している。
IPアドレス [Internet protocol address]
TCP/IPネットワーク上のコンピュータのアドレス (住所)。インターネット上のすべてのコンピュータはIPアドレスを持っている。クライアントは固定IPアドレスか、ネットワークに接続の都度動的に割り当てられるIPアドレスを持つ。IPアドレスは4つの数字の組、例えば 204.171.64.2 のようにして表される。
Java
サン マイクロシステムズ開発のプログラミング言語。単一のJavaプログラムは複数の異なる種類のコンピュータ上で動作するので、プログラマが各々別のバージョンを作成する必要がなくなる。&brandShortName;はJavaプログラムを自動でダウンロードして、実行できる (アプレットと呼ばれる)。
JavaScript
ウェブページ生成に良く用いられるスクリプト言語。プログラマがウェブページをもっとインタラクティブにするためにJavaScriptを用いる。例えばフォームやボタンの表示など。JavaScriptはJavaと共に使用することができるが、技術的には個別の言語。Javaが動作するのにJavaScriptは不要である。
PKCS #11 [public-key cryptography standard #11]
スマートカードなどのセキュリティデバイスを管理する、公開鍵暗号方式の標準。
SFTP [Secure Shell File Transfer Protocol]
→SFTP
SSL [Secure Sockets Layer]
認証済みの暗号化された接続を確立するための、クライアントとサーバ相互の認証が出来る通信手順。SSLはTCP/IP上、HTTP、LDAP、IMAP、NNTPなどの高位のネットワークの通信手順下で使用される。IETF (Internet Engineering Task Force) の定める標準仕様は TLS (Transport Layer Security) と呼ばれ、これはSSLを基礎としている。認証、暗号化も参照。
TLS
　→SSL
TCP/IP [Transmission Control Protocol/Internet Protocol]
様々なオペレーティングシステムが動作するコンピュータを接続するのに用いられる、UNIXの通信手順 (プロトコル)。TCP/IPはインターネットプロトコルの本質で、世界標準である。
URL [Uniform Resource Locator]
ウェブ上のファイルやその他リソースの場所を指定するために用いる、標準化されたアドレス。例えばhttp://www.mozilla.org/。URLを&brandShortName;のロケーションバーに入力することにより、ウェブページにアクセスすることが出来る。ウェブページ上のリンクから他のウェブページに移動するときにも、URLが用いられる。インターネットアドレス、ウェブアドレスとも呼ばれる。
URI [Uniform Resource Identifier]
ウェブ上のリソースを区別するための名前。URLもこれの一種。
W3C [World Wide Web Consortium]
ワールドワイドウェブコンソーシアム (W3C) とは、ウェブの標準勧告を作成する協議会である。W3Cによれば、「ワールドワイドウェブコンソーシアム (W3C) は、ウェブの可能性を最大限に導く、共同利用できる技術 (仕様、ガイドライン、ソフトウェア、および道具) を開発」する。W3Cの勧告には、例えばHTML 4.01、XHTML 1.1、CSS 2などがあり、これらはウェブ標準という呼び方をされる。
World Wide Web (WWW)
ウェブとも呼ばれる。ウェブサーバに格納され、(&brandShortName;などの) ウェブブラウザが表示するウェブページによって構成される、インターネットの一部。
XML [Extensible Markup Language]
データ表現に用いられる公開された標準。HTMLとは違い、XMLではウェブページ作成者が特殊なタグを定義することができる。詳しくはW3Cのオンライン文書、Extensible Markup Language (XML)を参照。
