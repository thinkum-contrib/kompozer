Preferencias de &brandShortName;

&brandShortName; tiene varios ajustes importantes en el panel de preferencias. Los usuarios pueden establecer las preferencias dependiendo de sus requisitos y &brandShortName; las tendrá en cuenta cada vez que el usuario lo ejecute.

Esta sección describe las opciones en el panel de preferencias de &brandShortName;. Si todavía no lo está viendo, siga estos pasos:

  1. Pulse en el menú Herramientas > Preferencias (Menú Mozilla en MacOS X) para abrir el panel de preferencias (Opciones).
  2. Haga clic en la categoría de &brandShortName; para ver los ajustes correspondientes disponibles para personalizació.

Para información sobre las opciones de publicación de &brandShortName;, vea la sección Opciones de publicación.

General

Las preferencias generales le permiten especificar ajustes al guardar los archivos y editar tablas. Estos ajustes se aplican a todos los documentos que cree.

Si no está viendo ya las preferencias generales de &brandShortName;, siga estos pasos:

  1. Abra el menú Editar (Menu de Mozilla en MacOS X) y elija Preferencias.
  2. Pulse en la categoría General del panel lateral.

Menú de páginas recientes

Número máximo de páginas listadas: especifique el número máximo de páginas que aparecerán en Páginas recientes dentro del menú Archivo.

Al guardar o publicar páginas

  • Mantener el formato del código fuente original: esta opción mantiene los saltos de línea y el formato original del código fuente HTML de la página. Selecciónelo si desea mantener los espacios (líneas extras, tabulaciones, etc.) que hacen el código fuente HTML más legible. Esta preferencia no afecta a cómo se verán sus páginas en la ventana del navegador.
  • Reformatear (Pretty print) el código HTML:: esta opción cambia el formato del código fuente HTML para hacerlo más legible, insertando saltos de línea y sangrados. Esta preferencia no afecta a cómo se verán sus páginas en la ventana del navegador.
  • Guardar imágenes y otros archivos asociados al guardar las páginas: si está marcado, todas las imágenes, JavaScript (JS), hojas de estilo en cascada (CSS), y otros archivos asociados se guardarán en la misma ubicación que el documento cuando éste sea guardado por primera vez o cuando sea guardado en una nueva ubicación. Si está desmarcado, sólo se guarda el archivo HTML.

  • Por ejemplo, al editar una página remota, esta opción asegura que todos los archivos relacionados asociados con la página remota sean guardados localmente cuando guarde la página a su disco duro.

  • Mostrar siempre el diálogo Publicar al publicar las páginas: si está marcado, &brandShortName; muestra siempre el cuadro de diálogo Publicación de página cuando pulse el botón Publicar o elija Publicar desde el menú Archivo. Si no está marcado, &brandShortName; muestra el diálogo Publicación de página solamente si necesita más información para publicarla.

Edición de tablas

Mantener el diseño de la tabla al introducir o borrar celdas: seleccione esta opción si quiere que &brandShortName; preserve siempre el diseño de la tabla (es decir, mantenga su forma rectangular) agregando celdas donde sea necesario. Si desmarca esta opción, cuando suprima una o más celdas, &brandShortName; eliminará también el borde de la celda, lo que puede dar lugar a una tabla con espacios vacíos, o un contorno irregular debido a un número desigual de celdas.

[Volver al principio de la sección]

Edición de hojas de estilo en cascada (CSS)

Usar estilos CSS en lugar de elementos y atributos HTML: habilita el uso de formato mediante hojas de estilo en cascada (CSS) en sus documentos de &brandShortName;. Con esta preferencia activada, &brandShortName; genera el formato de los elementos HTML 4.01 mediante estilos en línea CSS.

Si no se activa esta preferencia, &brandShortName; genera formato conforme con HTML 4.01, pero no utiliza estilos CSS.

Comparado con HTML, el HTML con formato CSS es más portable, más mantenible y más compatible cuando se visualiza con distintos navegadores. Si activa esta preferencia y después edita un documento creado sin CSS, &brandShortName; reemplaza los elementos editados por estilos CSS.

Si activa el uso de estilos CSS, puede elegir un color de resaltado de texto para el texto seleccionado usando el botón de color de resaltado de texto en la barra de formato. Puede también elegir un color de fondo para cualquier elemento de la página (estas características no están disponibles si esta preferencia no está activada).

Administrador de sitios

Abrir siempre un documento en una pestaña nueva: seleccione esta opción si desea abrir los documentos en nuevas pestañas mientras usa el administrador de sitios. Si no está marcado, al hacer doble clic en los nombres de archivo en el administrador de sitios, éstos se abrirán en nuevas ventanas.

Tipos de letra

Los tipos de letra mostrados en &brandShortName; son los predeterminados en el programa o los que el usuario defina por sí mismo.

Tipos de letra para

La familia tipográfica se puede definir en la sección Tipos de letra para: de las preferencias de &brandShortName;. Para cambiar la tipografía predeterminada:

  1. Desde la lista desplegable Tipos de letra para, elija un juego de caracteres. Por ejemplo, para fijar la tipografía por omisión para el juego de caracteres occidental (Romano), elija Occidental.
  2. Seleccione si el texto proporcional debe ser serif (como "Times New Roman") o sans-serif (como "Arial"). Luego especifique el tamaño de letra que desea para el texto proporcional.
  3. Especifique el tipo de letra a usar para las familias tipográficas Serif, Sans-serif, Cursive Fantasy y Monospace. Puede también cambiar el tamaño para los tipos de letra Monospace .

Puede también elegir el Tamaño de letra mínimo que desea mostrar. Fijar un tamaño de letra mínimo puede ser provechoso al editar páginas con tipografís muy pequeñas que serían difíficiles de leer si no se indica esta opción.

También se puede especificar la resolución de la pantalla, que es normalmente 96 ppp. Puede fijarlo a una de las opciones predefinidas p.e. 72ppp, o 96ppp o puede hacerlo coincidir con la configuración del sistema o establecerlo manualmente usando la opción Otro…

Config. nuevas páginas

La configuración para nuevas páginas permite que especifique los ajustes para colores e imágenes de fondo que se aplicarán a cada documento que cree.

Si no está viendo ya la configuración de nueva página, siga estos pasos:

  1. Abra el menú Herramientas (menú Mozilla en Mac OS X) y elija Preferencias.
  2. Pulse en el icono Config. nuevas páginas en el panel lateral de categorías

Autor

Introduzca su nombre. Esto agregará su nombre al código fuente HTML en cada página nueva que cree. Su nombre se añadirá en la sección <head> de la página web y no será visible.

[Volver al principio de la sección]

Marcado

  • Lenguaje: los usuarios pueden seleccionar el lenguaje a usar por omisión al crear un nuevo documento web. Elija HTML4.0 ó XHTML1.0.
  • DTD: seleccione el DTD que se usará de modo preterminado al crear nuevos documentos web. Están disponibles las opciones de definir el documento como Transitional o Strict.
[Volver al principio de la sección]

Apariencia predeterminada de la página

  • Colores predeterminados del usuario: seleccione esta opción si quiere que sus páginas usen siempre los ajustes de color del navegador de los visitantes para los elementos de texto y enlaces.
  • Usar colores personalizados: escoja esta opción si quiere especificar siempre los colores que se aplican a los elementos de texto y enlaces. Luego, para cada elemento seleccione un color pulsando el botón de color junto a cada elemento, lo que abrirá la paleta de colores para seleccionar el color fácilmente.
  • Imagen de fondo: escriba la ubicación y el nombre de la imagen, o pulse en Elegir Archivo para localizar la imagen en su disco duro o red local.

    Nota: las imágenes de fondo se aplican en mosaico y tienen preferencia sobre el color de fondo.

  • Para cambiar el nombre del autor de una página individual: abra el menú Formato y elija Título y propiedades de la página.

    Para cambiar los colores de la página y la imagen de fondo de una página individual: abra el menú Formato y elija Colores y fondo de la página.

[Volver al principio de la sección]

Internacionalización

  • Idioma: elija el idioma en el cual está previsto escribir el documento HTML.

  • Por ejemplo si el documento se escribe en francés canadiense, al seleccionar Francés/Canadá[fr-ca] indicará al navegador que el documento está escrito en francés canadiense. Y el código HTML contendrá el <html lang="fr-ca"> como indicador del idioma del documento.

  • Dirección de escritura: escoja la dirección de escritura del texto dependiendo del idioma que está usando en su documento. Esto solventa problemas que los usuarios tenían al crear páginas html en idiomas escritos de derecha a izquierda, por ejemplo, Árabe, Urdu, etc.

  • Para cambiar la especificación de idioma para una página individual: abra el menú Formato y elija Título y propiedades de página, y cambie el idioma en la sección Internacionalización.

    Para cambiar la dirección de escritura de una página individual: abra el menú Formato y elija Título y propiedades de página, y elija el idioma en la sección Internacionalización.

  • Conjunto de caracteres: el conjunto de caracteres determina cómo los bytes que representan el texto de su documento HTML se traducen a caracteres legibles. Con esta opción puede elegir el conjunto de caracteres que desea utilizar para su página web. Se usa el conjunto de caracteres ISO-8859-1 por omisión, pero puede usar otras opciones como UTF-8, ISO-8859-15, etc o usar un conjunto de caracteres definido por el usuario.

Configuración avanzada

Conexión

Esta sección describe cómo usar el panel de preferencias de proxies. Si no lo está viendo ya, siga estos pasos:

  1. Abra el menú Editar (el menú de &brandShortName; en Mac OS X) y elija Preferencias.
  2. En la categoría Avanzado, pulse en Configuración de la conexión (si no hay subcategorías visibles, haga doble-clic en Avanzado para expandir la lista).

El panel de las preferencias de proxies permite que configure &brandShortName; para utilizar un proxy:

Antes de empezar: pregunte a su administrador de red si tiene un archivo de la configuración del proxy o los nombres y números de puertos del proxy.

  • Conexión directa a Internet: elija esta opción si no quiere usar un proxy.
  • Configuración manual del proxy: elija esta opción si no tiene una dirección del proxy (URL).
    • Proxy HTTP, Proxy SSL, Proxy FTP, Proxy Gopher: escriba el nombre o número de dirección IP del servidor del proxy. Escriba el puerto en el campo Puerto.
    • Host SOCKS: escriba el nombre o dirección IP del servidor del proxy. Escriba el puerto en el campo Puerto.
      • SOCKS v4, SOCKS v5: Al introducir un host SOCK, seleccione "SOCKS v4" o "SOCKS v5", dependiendo de qué versión de SOCKS se use en el proxy.
    • No usar proxy para: escriba los dominios y/o direcciones IP para las que no quiere usar proxy. Separe las entradas con una coma (ejemplo: .suempresa.com, .suempresa.com.es, 192.168.1.0/24)
  • URL para configuración automática del proxy: elija esta opción si tiene una configuración de un proxy en un archivo o URL, y seguidamente escriba la URL de configuración.

Comportamiento de la tecla Enter

La opción Dentro de un párrafo, pulsar Enter lo cierra y crea uno nuevo define el comportamiento de la tecla de retorno de carro. Si está marcada y pulsa la tecla Enter mientras escribe dentro de un párrafo se creará uno nuevo. Sin esta opción, se añaden etiquetas <br> a modo de saltos de línea.

Comprobación ortográfica en tiempo real

Activar la opción Subrayar las palabras mal escritas permitirá la comprobación otrográfica en tiempo real en &brandShortName; para subrayar las palabras mal escritas, dando así al usuario una pista visual sobre dónde comprobar la otrografía.

Caracteres especiales

Estas opciones permiten al usuario elegir qué caracteres especiales son escritos como entidades en &brandShortName;. Elija una de las opciones:

  • Sólo & < > ' y espacios blancos no-rompibles
  • Los anteriores y las letras Latin-1
  • Todos los que tengan una entidad equivalente en la espec. HTML 4
  • Usar notación &#..; para todos los caracteres no ASCII
